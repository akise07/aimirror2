## 参考图片打包进应用
- 将 `ref/` 下 17 张参考图片复制到 `public/ref/`，Vite 会自动将 public 目录内容打包进静态资源
- 修改 `api.ts` 中 `getRefImages()`：从 `${LOCAL_BASE}/ref/${name}` 改为 `/ref/${name}`（静态资源路径）

## 页面切换保持生成状态
- App.vue 中 `<router-view>` 用 `<keep-alive>` 包裹，切换页面时组件不销毁，轮询/流式读取继续运行
- 移除 `<transition>` 动画（与 keep-alive 组合有兼容性问题）
- MakeupGen / VideoGen / RecommendView 三个组件均添加 AbortController 取消机制
  - 轮询循环每轮检查 signal.aborted，sleep 也支持 AbortSignal
  - resetTask() / resetRecommend() 调用 controller.abort() 终止循环
  - 防止重置后旧循环继续运行导致状态错乱

## video_gen 接口改为 FormData 上传
- 后端 `video_gen` 接口从 JSON(`image_url`+`text`) 改为 FormData(`image_id` 文件+`text` form字段)
- `api.ts`：`createVideoTask(imageUrl, text)` → `createVideoTask(imageId: File, text)`
- `VideoGen.vue`：选中图片后用 `urlToFile()` 将图片 fetch 为 File 对象，再通过 FormData 上传
- 与 `makeup_image` 接口保持一致的文件上传方式
