## 拍照缓存修复
- 问题：实时预览页面拍照缓存不工作，`canvas.toDataURL()` 因跨域 canvas 污染静默失败
- 原因：MJPEG 流来自跨域 `http://192.168.137.151:3401/video_feed`，Flask 无 CORS 头，`<img>` 无 crossorigin 属性
- 修复：
  - Flask 后端 `api.py` 添加全局 CORS 头 (`Access-Control-Allow-Origin: *`)
  - `document.querySelector` 改为模板 `ref`（更可靠）
  - 添加 try-catch 错误处理和拍照反馈消息
  - 使用 `naturalWidth/naturalHeight` 替代硬编码 1024
- ⚠️ 踩坑：`crossorigin="anonymous"` 不能用于 MJPEG 流！
  - Electron 已设 `webSecurity: false`，canvas 不会被污染，不需要 crossorigin
  - crossorigin 强制浏览器对 MJPEG 流做 CORS 校验，会导致流加载失败（显示"摄像头未连接"）
  - 已移除 crossorigin 属性

## 拍照截帧改用 fetch 方案
- 问题：去掉 crossorigin 后，视频流恢复正常，但 canvas 依然被污染（Tainted canvas）
  - `webSecurity: false` 绕过了 fetch/XHR 跨域，但**不能阻止 canvas tainting**
- 方案1（已废弃）：fetch 后端 `/video_frame` 单帧接口
  - Flask 新增了 `/video_frame` 端点，`api.ts` 新增了 `getVideoFrameUrl()`
  - 但后端运行在远程设备(192.168.137.151)，需要手动部署重启才能生效
  - fetch 不存在的端点报 "Failed to fetch"
- 方案2（当前方案）：前端直接从 MJPEG 流解析 JPEG 帧
  - `fetch(getVideoFeedUrl())` 获取 MJPEG 流的 ReadableStream
  - 读取数据直到找到 JPEG 起始标记(FFD8)和结束标记(FFD9)
  - 提取 JPEG bytes → Blob → FileReader → base64
  - 完全不需要后端改动，不需要 canvas
