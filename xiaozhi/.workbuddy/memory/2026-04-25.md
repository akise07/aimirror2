- 兼容两种消息格式：直接 JSON-RPC 和 `{"type":"mcp","payload":{...}}` 包裹格式
- 注册了两个工具：

- 协议交互流程：initialize → notifications/initialized → tools/list → tools/call
- 新增：对话消息打印功能（handle_conversation_message），支持 STT/TTS/LLM/listen/text/abort 等消息类型
- 当 MCP 连接收到非 JSON-RPC 的对话消息时，自动提取并打印用户说的内容和AI的回答

## 重构 xiaozhi_mcp.py（对话消息监听器）
- 重写为 `xiaozhi_mcp.py`，作为MCP客户端连接小智AI，监听并打印对话消息
- **协议握手流程**（双向initialize）：
  1. 客户端发送 initialize → 服务器也发送自己的 initialize（双向）
  2. 客户端响应服务器的 initialize
  3. 服务器发送 notifications/initialized
  4. 服务器发送 tools/list（请求客户端的工具列表）→ 客户端响应
  5. 握手完成，进入消息监听
- **关键发现**：小智MCP协议中，服务器（xz-mcp-broker）是"host"角色，会主动向客户端发请求（initialize、tools/list、ping）
- 支持消息类型：stt（用户语音转文字）、tts（AI语音）、llm（AI文本回复）、listen、text、abort、emotion、iot、notification
- 解决了 Windows GBK 编码问题（设置 stdout UTF-8）
- 创建了 test_mcp.py 测试脚本，验证连接和握手流程正常
